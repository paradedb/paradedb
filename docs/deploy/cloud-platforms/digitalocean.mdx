deploying ParadeDB on a DigitalOcean Droplet using Docker. Docker packages PostgreSQL and `pg_search` together, so you don't need to install them manually.

1. A DigitalOcean account

## Create a Droplet

Create a new Droplet from the DigitalOcean dashboard with the following settings:

- **Image**: Ubuntu 24.04 (LTS) x64
- **Size**: See the [DigitalOcean plan guide](https://docs.digitalocean.com/products/droplets/concepts/choosing-a-plan/) for sizing recommendations

Once the Droplet is running, SSH into it to begin setup.

## Configure Environment Variables

Set the following variables to match your desired Postgres and ParadeDB versions. Replace `<YOUR_IP_ADDRESS>` with your
local machine's IPv4 address (not the Droplet IP).

```bash
PG=18
VERSION="0.21.10"
ALLOWED_CIDR="<YOUR_IP_ADDRESS>/32"
DIST=noble
ARCH="$(dpkg --print-architecture)"
```

## Configure Postgres

Configure Postgres to accept remote connections from your client IP:

```bash
sudo sed -i "s/^#\?listen_addresses\s*=.*/listen_addresses = '*'/" \
  "/etc/postgresql/${PG}/main/postgresql.conf"
sudo sed -i "s/^#\?port\s*=.*/port = 5432/" \
  "/etc/postgresql/${PG}/main/postgresql.conf"

echo "host all all ${ALLOWED_CIDR} scram-sha-256" \
  | sudo tee -a "/etc/postgresql/${PG}/main/pg_hba.conf"

sudo systemctl restart postgresql
sudo systemctl enable postgresql
```

<Note>
  This allows only the IP address specified in `ALLOWED_CIDR`. To whitelist
  additional IP addresses, add them to `/etc/postgresql/{PG}/main/pg_hba.conf`.
</Note>

Verify that Postgres is running:

```bash
pg_lsclusters
```

If the status is not `online`, check the logs:

```bash
sudo tail -n 100 /var/log/postgresql/postgresql-${PG}-main.log
```

## Configure Firewall

In the DigitalOcean console, navigate to **Droplets** → your Droplet → **Networking** and create a firewall with the
following inbound rule:

| Field    | Value                  |
| -------- | ---------------------- |
| Type     | Custom                 |
| Protocol | TCP                    |
| Port     | 5432                   |
| Sources  | `<YOUR_IP_ADDRESS>/32` |
