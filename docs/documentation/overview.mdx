## Welcome to the Default API

This section documents ParadeDB's current SQL interface.
The `v2` API reached feature parity with `v1` and is now the default.

If you need the older interface reference, use `Legacy Docs` in the left navigation bar.

Existing `v1` integrations remain supported, so there is no forced migration.
The current API focuses on improving developer experience across three key areas:

### Declarative Schema Configuration

No more complex JSON strings to configure tokenizers, fast fields, etc.

### Transparent Search Operators

More intuitive syntax and more transparent behavior, making it easier to understand exactly what's happening just by reading the SQL query.

### ORM-Friendly Query Builders

Queries will support structured column references instead of relying on string literals, improving compatibility with query builders and reducing runtime errors.

## What Changed

This documentation section covers the newer interface going forward.
For older examples, migration work, or long-lived integrations that still use the previous syntax, refer to `Legacy Docs`.
