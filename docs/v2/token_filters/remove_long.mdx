---
title: Remove Long
---

The remove long token filter automatically excludes any token longer than a fixed number of bytes from the index.
By default, tokens longer than `255` bytes are excluded for all tokenizers besides the [exact](/v2/tokenizers/available_tokenizers/exact)
tokenizer.

```sql
CREATE INDEX search_idx ON mock_items
USING bm25 (id, (description::pdb.simple('remove_long=100')))
WITH (key_field='id');
```

To demonstrate this token filter, let's compare the output of the following two statements:

```sql
SELECT
  'A supersupersuperlong token'::pdb.simple::text[],
  'A supersupersuperlong token'::pdb.simple('remove_long=10')::text[];
```

```ini Expected Response
             text              |   text
-------------------------------+-----------
 {a,supersupersuperlong,token} | {a,token}
(1 row)
```
